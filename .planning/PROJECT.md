# AgentSafeEnv

## What This Is

A multi-agent orchestration platform that deploys, monitors, and controls AI coding agents (Claude, Copilot, Codex, Gemini, OpenCode — with OpenClaw and NanoClaw later) across a fleet of 5-10 machines. It provides a unified control plane for launching agent sessions, streaming real-time output and status, and managing host inventory — whether agents work independently on separate tasks or collaborate on shared codebases.

## Core Value

See what every agent is doing across every machine in real-time, and be able to deploy new agent sessions to any registered host from one place.

- OpenClaw / NanoClaw agent support — future iteration
- Tool installation on hosts during registration — needed for final version, not v1
- Mobile-optimized MAUI UI — desktop-first
- Multi-tenant / multi-user auth — single operator for now

## Context

An existing scaffold (generated by ChatGPT) provides the project structure with ASP.NET Core service, Aspire AppHost, MAUI client skeleton, and orchestration abstractions. All code should be treated as first draft — review and refine everything. The scaffold has useful architectural patterns (backend abstraction, skill/policy config, sanitization) worth preserving in spirit, but implementations are stubs.

Target machines already have some agent CLIs installed. Agents are launched via their respective CLI commands (e.g., `claude`, `codex`, `gh copilot`). The platform wraps and orchestrates these rather than replacing them.

The user has experience with vibe-kanban (worktree-based workflows) and GSD plugin patterns. MCP tooling is interesting as a control plane bridge to existing agent sessions but is one protocol among several, not the only one.

## Constraints

- **Stack**: .NET 10 / ASP.NET Core / Aspire — primary technology, non-negotiable
- **Platform**: Windows-first, but should work cross-platform where .NET supports it
- **Agents**: Must work with agents as-they-are (CLI-based tools), not require agents to be modified
- **Safety**: Keep flexible policy/skill/sanitization abstractions from scaffold — will be needed for incremental hardening
- **Data**: EF Core with provider abstraction — start with simplest option, swap later
