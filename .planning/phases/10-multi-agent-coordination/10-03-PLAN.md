  <action>
    1. **CLI - AgentHubApiClient**: Ensure the session list API response deserializes ParentSessionId from the JSON. The SessionSummary record already has ParentSessionId -- verify the JSON property name matches.

    2. **CLI - SessionListCommand**: After fetching sessions, group them into a tree structure:
       - Separate root sessions (ParentSessionId is null) from child sessions.
       - Create a Spectre.Console `Tree` with label "[bold]Sessions[/]".
       - For each root session, add a tree node with: truncated session ID (first 8 chars), state (color-coded: green=Running, dim=Stopped, red=Failed), host node name, agent type.
       - For each child of that root (matching ParentSessionId), add a child node with same formatting but dimmer styling.
       - Support nested children (grandchildren) by recursive traversal (max depth 3 per config).
       - If no parent-child relationships exist, fall back to the existing flat table display to avoid visual regression.
       - Use `AnsiConsole.Write(tree)` to render.

    3. **Web - DashboardApiClient**: Ensure GetSessionsAsync response model includes ParentSessionId. Add a method or param for fetching children: `GetSessionsAsync(string? parentId = null)` that passes `?parentId=` query param.

    4. **Web - SessionTable.razor**: Modify the session table/list to show hierarchy:
       - Group sessions: root sessions (no parent) shown as primary rows.
       - Each root session with children gets an expand/collapse toggle (MudIconButton with chevron).
       - When expanded, child sessions appear indented below the parent with a subtle left border or background tint for visual distinction.
       - Implementation approach: use `@foreach` with conditional rendering. Keep it simple -- a CSS margin-left for indentation, a thin left border accent.
       - Child sessions clicking through to detail should work exactly as standalone sessions.
       - Show a small badge or icon on parent rows indicating child count (e.g., "[2 children]" chip).
       - Alternatively, use MudTreeView if it fits naturally with the existing table layout. Use your judgment on which approach integrates better with the existing SessionTable component.

    5. Build verification: `dotnet build` for both CLI and Web projects.
