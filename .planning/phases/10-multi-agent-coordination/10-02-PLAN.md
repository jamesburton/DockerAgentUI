  <behavior>
    - When a child session emits a ChildCompleted/ChildFailed/ChildSpawned/StateChanged event, it is forwarded to the parent's SSE channel
    - Forwarded events have [child-session-id] prefix in Data field
    - Forwarded events have meta tag ["forwarded"] = "true" to prevent infinite loops
    - Events already marked as forwarded are NOT re-forwarded (loop prevention)
    - SSE session subscribe endpoint accepts ?includeChildren=true query param
    - When includeChildren=true, ALL child events (not just lifecycle) are forwarded to parent stream
    - Parent-child ID mapping is cached in ConcurrentDictionary for fast lookup (no DB query per event)
    - Cache is invalidated when session completes/fails
  </behavior>
  <action>
    1. Add a parent-child cache to DurableEventService: `ConcurrentDictionary<string, string?> _parentCache` (childId -> parentId). Populate lazily on first lookup from DB. Provide method InvalidateParentCache(sessionId) called when session stops/fails.

    2. In DurableEventService.EmitAsync, AFTER the existing broadcast logic, add parent forwarding:
       - Skip if event meta contains ["forwarded"] = "true" (loop prevention).
       - Look up parent ID from cache (or DB on miss).
       - If parent exists:
         - Determine if event is lifecycle: Kind is ChildSpawned, ChildCompleted, ChildFailed, or StateChanged.
         - Create prefixed event: `ev with { SessionId = parentId, Data = $"[{ev.SessionId}] {ev.Data}", Meta = new Dictionary<string,string>(ev.Meta ?? new()) { ["forwarded"] = "true", ["sourceSessionId"] = ev.SessionId } }`.
         - Call `_subs.BroadcastToParent(parentId, prefixedSseItem, lifecycleOnly)` where lifecycleOnly = !includeChildren for that subscriber.

    3. Update SseSubscriptionManager:
       - Change the session subscriber dictionary value type to include a `bool IncludeChildren` flag. Use a wrapper: `record SubscriberInfo(ChannelWriter<SseItem<SessionEvent>> Writer, bool IncludeChildren)`.
       - Update AddSessionSubscriber to accept includeChildren parameter.
       - Add `BroadcastToParent(string parentSessionId, SseItem<SessionEvent> sseItem, bool isLifecycleEvent)` method:
         - Looks up subscribers for parentSessionId.
         - For each subscriber: if isLifecycleEvent, always forward. If not lifecycle, only forward if subscriber.IncludeChildren is true.

    4. Update DurableEventService.SubscribeSession to accept and pass through includeChildren parameter.

    5. In Program.cs SSE endpoint `GET /api/sessions/{sessionId}/events`, add `bool? includeChildren` query param (default false). Pass to SubscribeSession.

    6. When a child session completes (StateChanged to Stopped) or fails (StateChanged to Failed), emit ChildCompleted or ChildFailed event on the parent's stream. This can be done in the parent forwarding logic: if the child's event is StateChanged with data "Stopped", also emit ChildCompleted to parent. If "Failed", emit ChildFailed.

    7. Write ChildEventForwardingTests.cs with [Trait("Category", "Phase10")]:
       - Test: child lifecycle event is forwarded to parent's channel
       - Test: child non-lifecycle event is NOT forwarded when includeChildren=false
       - Test: child non-lifecycle event IS forwarded when includeChildren=true
       - Test: forwarded event has [child-id] prefix and forwarded=true meta
       - Test: already-forwarded event is NOT re-forwarded (loop prevention)
       - Test: parent cache invalidation works
