  <behavior>
    - PlacementScoringTests:
      - ScoreNode returns highest score for host with most available CPU/memory/lowest sessions
      - Host with stale metrics (>2x polling interval of 30s) gets score -1 (excluded)
      - Host with no metrics snapshot returns -1
      - Host with disk below MinDiskFreeGb returns -1
      - Host at MaxSessionsPerHost returns -1
      - TargetHostId bypasses scoring and returns that host directly
      - With two hosts, the higher-scored host is selected
    - CascadeLimitTests:
      - Depth 0 parent allows spawn (depth < MaxDepth)
      - Depth at MaxDepth rejects with InvalidOperationException containing "depth"
      - Parent with MaxChildrenPerParent active children rejects with InvalidOperationException containing "children"
      - Parent with fewer than max children allows spawn
  </behavior>
  <action>
    1. Upgrade SimplePlacementEngine constructor to accept IOptions<PlacementOptions>, HostMetricCache, and ActiveSessionTracker via DI.

    2. Replace the ChooseNode logic:
       - Keep ALL existing filter logic (ExecutionMode, TargetHostId, Os, NeedsGpu, CpuMin, MemMinMb, AcceptRisk, Labels).
       - If TargetHostId is set and a matching node is found, return it immediately (bypass scoring).
       - For remaining candidates, call ScoreNode() for each. Filter out scores < 0. OrderByDescending score. Return top result.
       - If no candidates remain after scoring, throw InvalidOperationException("No eligible node found").

    3. Implement private ScoreNode method following the research pattern:
       - Get HostSnapshot from HostMetricCache by nodeId. No snapshot = return -1.
       - Stale check: if MetricsUpdatedUtc is older than 60 seconds from now, return -1.
       - Disk hard filter: if DiskFreeGb < MinDiskFreeGb, return -1.
       - CPU score: (100 - CpuPercent) / 100, default 0.5 if null.
       - Memory score: (MemTotal - MemUsed) / MemTotal, default 0.5 if null.
       - Session score: 1.0 - (activeSessions / MaxSessionsPerHost). If activeSessions >= MaxSessionsPerHost, return -1.
       - Composite: cpu * CpuWeight + mem * MemoryWeight + session * SessionWeight.
       - Stale penalty: if metrics age > 30s (but <= 60s), multiply by StaleMetricPenalty.

    4. Add ValidateCascadeLimitsAsync method to SessionCoordinator (or as a separate static/helper - keep it testable). Uses IDbContextFactory and IOptions<CoordinationOptions>:
       - GetSessionDepthAsync: iterative walk of ParentSessionId chain. Max iterations = MaxDepth + 1 (safety).
       - Check depth: if parentDepth >= MaxDepth, throw InvalidOperationException with depth info.
       - Check child count: COUNT sessions WHERE ParentSessionId = parentId AND State = Running. If >= MaxChildrenPerParent, throw with count info.

    5. Inject IOptions<CoordinationOptions> and IDbContextFactory into SessionCoordinator (dbFactory already injected). DO NOT wire ValidateCascadeLimits into StartSessionAsync yet -- Plan 02 will do that when adding spawn flow.

    6. Update DI registration in Program.cs: SimplePlacementEngine constructor now needs IOptions<PlacementOptions>, HostMetricCache, ActiveSessionTracker. Adjust registration if needed (it may already be registered as IPlacementEngine).
