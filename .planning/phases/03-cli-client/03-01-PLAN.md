  <action>
    **All commands use System.CommandLine 2.0 stable API** (CliRootCommand, CliCommand, CliArgument, CliOption, SetAction). NOT the beta API (RootCommand, Command, Option, SetHandler).

    **Program.cs** - Build the full command tree:
    1. Create CliRootCommand with description "AgentHub CLI - manage agent sessions across your fleet"
    2. Add global options: `--json` (bool), `--no-color` (bool), `-v`/`--verbose` (bool), `-q`/`--quiet` (bool), `--server` (string, override server URL)
    3. Create service instances: `CliConfig.Load()`, `new HttpClient { BaseAddress = serverUrl }`, `new AgentHubApiClient(httpClient)`, formatter based on --json flag
    4. Add command groups: `session` (with subcommands: start, list, stop), `host` (with subcommands: list, status), `config` (with subcommand: show)
    5. Add top-level shortcuts: `run` (same action as session start), `ls` (same action as session list)
    6. Handle NO_COLOR env var: if set, force --no-color behavior
    7. Set `parseResult.Configuration.ExitCode` on errors. Wrap all command actions in try/catch: HttpRequestException -> exit 1, OperationCanceledException -> exit 2
    8. Call `new CliConfiguration(rootCommand).InvokeAsync(args)` to run

    **Each command is a static factory method** returning a CliCommand (not a class per se, but a builder method). Use a static class per resource group (e.g., `SessionCommands.CreateStartCommand(...)`) that takes ApiClient and formatter as parameters. This keeps commands testable and avoids DI complexity in a CLI.

    **SessionStartCommand.cs** (`ah session start <agent> <prompt>` / `ah run`):
    - Arguments: `agent` (string, positional), `prompt` (string, positional or --prompt/-p)
    - Options: `--host` (string, optional -- maps to TargetHostId), `--detach/-d` (bool, default false), `--fire-and-forget` (bool)
    - Action: Build StartSessionRequest with ImageOrProfile=agent, Prompt=prompt, Requirements with TargetHostId=host. Call apiClient.StartSessionAsync(). On success, formatter.WriteSuccess with session ID. If NOT --detach, print "Attached to session {id}. Use Ctrl+C to detach." (actual live streaming is Plan 02; for now just print the message and exit).
    - Exit code: 0 on success, 1 on API error

    **SessionListCommand.cs** (`ah session list` / `ah ls`):
    - Options: `--state` (string filter), `--take` (int, default 20)
    - Action: Call apiClient.GetSessionsAsync(). Render table with columns: ID (first 8 chars), State (color-coded: green=Running, red=Failed, grey=Stopped, yellow=Pending), Agent (from Requirements.ToString() or ImageOrProfile), Host (Node or "(auto)"), Created (local HH:mm:ss), Duration.
    - In verbose mode (-v): show full ID, backend, and Requirements details

    **SessionStopCommand.cs** (`ah session stop <id>`):
    - Arguments: `sessionId` (string, positional)
    - Options: `--force/-f` (bool)
    - Action: Call apiClient.StopSessionAsync(sessionId, force). formatter.WriteSuccess("Session {id} stopped").

    **HostListCommand.cs** (`ah host list`):
    - Action: Call apiClient.GetHostsAsync(). Render table with columns: ID, Name, OS, Backend, Enabled, SSH.
    - Verbose: add Address, Labels columns.

    **HostStatusCommand.cs** (`ah host status`):
    - Options: `--watch` (bool, default false -- actual live refresh is Plan 02; for now just print once)
    - Action: Call apiClient.GetHostsAsync(). Render table with columns: Name, OS, CPU% (from CpuPercent, show "--" if null), Mem (MemUsedMb/MemTotalMb, show "--" if null), Sessions (placeholder "--" for now).
    - Color code: CPU > 80% red, > 60% yellow, else green. Mem > 90% red, > 70% yellow.

    **ConfigShowCommand.cs** (`ah config show`):
    - Action: Load CliConfig, display as a Spectre.Console Panel showing ServerUrl, DefaultHost, DefaultAgent, WatchRefreshMs, LogTailDefault. In --json mode, serialize the CliConfig object.
