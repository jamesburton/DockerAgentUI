Build the approval/elevation gating flow with configurable timeouts and the extended sanitization layer with trust-tier-based action classification.

Purpose: This is the safety layer of Phase 2. Without approval gating, destructive agent actions execute without human confirmation. Without extended sanitization, injection attacks could compromise remote hosts.

Output: ApprovalService with SSE-delivered approval requests, configurable timeout behavior, and race-condition-safe resolution. Extended BasicSanitizationService with shell injection, path traversal, and env var exfiltration detection, driven by trust tier config.

1. Create ApprovalService.cs in Coordinator/:
   - Inject `IDbContextFactory<AgentHubDbContext>`, `ILogger<ApprovalService>`
   - Register as singleton (needs ConcurrentDictionary for pending approvals)
   - `ConcurrentDictionary<string, TaskCompletionSource<ApprovalDecision>> _pending`

   - `enum ApprovalDecision { Approved, Denied, TimedOut, AutoApproved }`

   - `record ApprovalContext(string Action, string? Command, string? FilePath, string? DiffPreview, int? TimeoutSeconds, string? TimeoutAction, bool SkipPermissionPrompts)`

   - `async Task<ApprovalDecision> RequestApprovalAsync(string sessionId, ApprovalContext context, Func<SessionEvent, Task> emit, CancellationToken ct)`:
     a. If context.SkipPermissionPrompts == true: return AutoApproved immediately (no event, no DB record needed -- per user decision "auto-approve without prompting")
     b. Generate approvalId = Guid.NewGuid().ToString("N")
     c. Create TaskCompletionSource<ApprovalDecision>, store in _pending[approvalId]
     d. Persist ApprovalEntity to DB: Status="pending", TimeoutSeconds, TimeoutAction
     e. Emit SessionEvent with Kind=ApprovalRequest, Data=JSON-serialized ApprovalContext, Meta={"approvalId": approvalId}
     f. Create linked CancellationTokenSource: if TimeoutSeconds > 0, add timeout
     g. Try await tcs.Task.WaitAsync(cts.Token)
     h. On OperationCanceledException: determine timeout action from context.TimeoutAction:
        - "auto-approve" -> return Approved, update entity Status="approved"
        - "stop" -> return Denied, update entity Status="timed-out"
        - "continue" (default) -> return Approved, update entity Status="timed-out"
     i. Finally: _pending.TryRemove(approvalId, out _)

   - `void ResolveApproval(string approvalId, bool approved, string? resolvedBy)`:
     a. If _pending.TryGetValue(approvalId, out var tcs):
        - Use TrySetResult (not SetResult!) -- first to complete wins (race condition safety per research pitfall 3)
        - Update ApprovalEntity: Status = approved ? "approved" : "denied", ResolvedUtc, ResolvedBy
     b. If not found: log warning (already resolved or timed out)

2. Write ApprovalServiceTests.cs covering all behavior items:
   - Use EF Core InMemory provider
   - Use short timeouts (100ms) for timeout tests
   - For race condition test: start approval, fire resolve and cancel simultaneously, verify only one outcome
   - Verify ApprovalEntity status updates in DB after resolution
   - Verify SSE event emission (capture via mock emit function)
