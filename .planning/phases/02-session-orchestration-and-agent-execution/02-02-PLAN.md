Rewrite the stub SshBackend into a real SSH execution backend using SSH.NET, implement the two-phase stop mechanism, build the SessionMonitorService for heartbeat/orphan detection, and move session state from in-memory ConcurrentDictionary to database-backed queries.

Purpose: This is the core execution engine of Phase 2. Without real SSH execution, sessions are just database records with no actual agent running on remote hosts.

Output: Working SshBackend that connects to hosts via SSH.NET, sends HostCommandProtocol commands, streams events back, supports graceful and force stop, and persists all state to the database. SessionMonitorService detects orphaned sessions via heartbeat timeout.

  <name>Task 1: SshHostConnection wrapper and rewrite SshBackend with SSH.NET</name>
  <files>
    src/AgentHub.Orchestration/Backends/SshHostConnection.cs,

  <action>
1. Create SshHostConnection.cs -- a connection wrapper for a single SSH host:
   - Constructor takes: host address, username, private key path
   - Uses `Renci.SshNet.SshClient` with `PrivateKeyAuthenticationMethod`
   - Sets `KeepAliveInterval = TimeSpan.FromSeconds(15)` per research recommendation
   - Registers `client.ErrorOccurred` event handler to track connection state
   - `ConnectAsync(CancellationToken)` -- wraps _client.Connect() in Task.Run (SSH.NET is synchronous)
   - `ExecuteCommandAsync(string commandJson, CancellationToken)` -- creates SshCommand, wraps Execute in Task.Run, returns result string
   - `StartDaemonSessionAsync(string commandJson, CancellationToken)` -- for long-running sessions: uses SshClient.CreateCommand with stdin/stdout streaming. Sends command, returns (StreamReader stdout, StreamReader stderr) for event streaming.
   - `IsConnected` property
   - `IAsyncDisposable` -- disposes heartbeat timer and disconnects client
   - Internal heartbeat timer (30s interval per research discretion) sends ping commands, sets _connected=false on failure

2. Update ISessionBackend.StopAsync signature in Abstractions.cs to accept a `bool forceKill` parameter:
   `Task StopAsync(string sessionId, bool forceKill, CancellationToken ct);`
   Update InMemoryBackend and NomadBackend to accept the new parameter (ignore forceKill for InMemoryBackend, just stop).

3. Rewrite SshBackend.cs completely:
   - Inject `IDbContextFactory<AgentHubDbContext>`, `IHostRegistry`, `ILogger<SshBackend>`
   - Remove `ConcurrentDictionary<string, SessionSummary> _sessions` -- use DB for all state
   - Add `ConcurrentDictionary<string, SshHostConnection> _connections` -- tracks active SSH connections per session
   - `Name => "ssh"`
   - `CanHandle` -- same logic but also check host.AllowSsh from IHostRegistry
   - `GetInventoryAsync` -- query DB via IHostRegistry for hosts with Backend="ssh", map to NodeCapability
   - `StartAsync`:
     a. Create SshHostConnection to the target host (get address from IHostRegistry)
     b. Build StartSessionPayload from request (agentType from request.ImageOrProfile, prompt from request.Prompt or request.Reason, workingDirectory, permissions including SkipPermissionPrompts)
     c. Send start-session command via HostCommandProtocol over SSH
     d. Parse response -- if success, persist SessionEntity to DB with State=Running, IsFireAndForget from request
     e. Start background Task to read stdout line-by-line, parse each line as HostCommandResponse, map to SessionEvent, call emit()
     f. Track connection in _connections dictionary
     g. Return sessionId
   - `StopAsync(sessionId, forceKill, ct)`:
     a. If forceKill: send force-kill command immediately, update DB state to Stopped
     b. If graceful: send stop-session command (SIGINT), wait 10 seconds (configurable), check if still running, if so send force-kill (SIGTERM)
     c. Dispose SshHostConnection, remove from _connections
   - `GetAsync` -- query SessionEntity from DB, map to SessionSummary using EntityMappers
   - `ListAsync` -- query SessionEntities from DB filtered by ownerUserId, map to SessionSummary list
   - `SendInputAsync` -- send input as a command over the active SSH connection for the session

4. SSH key path: read from configuration (IConfiguration) with key "Ssh:PrivateKeyPath", default to "~/.ssh/id_rsa".
