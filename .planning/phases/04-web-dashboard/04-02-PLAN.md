       A MudDialog for launching sessions:
       - MudSelect for agent type (enumerate AgentType enum values as string names)
       - MudSelect for host (load hosts via injected DashboardApiClient, include "Auto-place" as empty string option)
       - MudTextField (Lines=3) for prompt text
       - MudCheckBox for "Fire and forget" (IsFireAndForget)
       - MudCheckBox for "Skip permissions" (AcceptRisk on Requirements)
       - Cancel and Launch buttons
       - On Launch: construct StartSessionRequest with:
         - ImageOrProfile = selected agent type string (e.g., "ClaudeCode")
         - Requirements = new SessionRequirements with TargetHostId (if host selected), AcceptRisk (if skip permissions), ExecutionMode = ExecutionMode.Ssh
         - Prompt = prompt text
         - IsFireAndForget = checkbox value
       - Call DashboardApiClient.StartSessionAsync, close dialog on success
       - Show error via ISnackbar on failure

    2. Create `src/AgentHub.Web/Components/Pages/FleetOverview.razor`:
       Route: @page "/"

       Inject: DashboardApiClient, SseStreamService, NavigationManager, IDialogService, ISnackbar

       State:
       - `List<HostRecord> _hosts`
       - `List<SessionSummary> _sessions`
       - `string? _selectedHostId` (null = all hosts)
       - `bool _sseEnabled` (default false)
       - `Timer? _pollingTimer`
       - `CancellationTokenSource? _sseCts`

       Layout (MudGrid):
       - Left column (MudItem xs=12 md=3): HostSidebar component bound to _hosts, _selectedHostId, filtered _sessions
       - Right column (MudItem xs=12 md=9):
         - Toolbar row: MudText "Sessions" title, MudSpacer, MudSwitch "Live Updates" bound to ToggleSse, MudButton "Launch Session" that opens LaunchDialog
         - SessionTable component bound to filtered sessions (if _selectedHostId is set, filter where Node == _selectedHostId)

       Data loading (LoadDataAsync):
       - Call GetHostsAsync and GetSessionsAsync in parallel
       - Update _hosts and _sessions state

       Polling:
       - OnInitializedAsync: call LoadDataAsync, then start Timer with 7-second interval
       - Timer callback: if !_sseEnabled, call LoadDataAsync + InvokeAsync(StateHasChanged)

       SSE toggle (ToggleSse):
       - When enabled: create CancellationTokenSource, call SseStreamService.SubscribeFleet, start reading events
         - For each event: find matching session in _sessions by SessionId, update its state if StateChanged event, or add if new
         - Call InvokeAsync(StateHasChanged) after each event
       - When disabled: cancel CTS, resume polling only

       Session actions:
       - OnSessionSelected: NavigationManager.NavigateTo($"/session/{sessionId}")
       - OnStopSession: call DashboardApiClient.StopSessionAsync, show ISnackbar success/error, reload data
       - LaunchSession: open LaunchDialog via IDialogService.ShowAsync, reload data after dialog closes

       Disposal:
       - Implement IDisposable
       - Dispose Timer and cancel SSE CTS
  </action>
