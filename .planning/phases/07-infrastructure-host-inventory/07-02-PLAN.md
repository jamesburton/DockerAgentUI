  <action>
    Create `src/AgentHub.Orchestration/Monitoring/HostInventoryPollingService.cs` following the exact HostMetricPollingService pattern:

    1. Sealed class extending BackgroundService with constructor taking:
       - IDbContextFactory<AgentHubDbContext>, ISshHostConnectionFactory, HostMetricCache, DurableEventService, ILogger, IConfiguration
       - Read poll interval from `IConfiguration["Inventory:PollIntervalMinutes"]` (default 60), SSH key/username from existing Ssh: config section
    2. ExecuteAsync: identical while-loop + RunOnceAsync + Task.Delay pattern
    3. Public `RunOnceAsync(CancellationToken ct)`: query enabled SSH hosts from DB, call PollHostAsync per host with per-host try/catch for fault isolation
    4. Public `RefreshHostAsync(string hostId, CancellationToken ct)`: single-host refresh for on-demand API
    5. Static `GetInventoryCommand(string os, List<AgentConfig> agents)`:
       - Read agents from agents.json (loaded on each poll cycle per research recommendation)
       - Build composite command per OS:
         - **Windows**: PowerShell command that loops agent names with `where.exe`, captures versions with versionFlag, gets disk via `Get-PSDrive C`, gets git version. Returns JSON via `ConvertTo-Json -Compress`. Use `||` equivalent (try/catch per agent) so one missing agent doesn't kill the probe.
         - **Linux**: bash command using printf/echo for JSON construction (NOT jq -- avoid dependency per research open question). Each agent probe is independent with `|| true`. Disk via `df -BG /`. Git via `git --version`.
         - **macOS**: similar to Linux but with `df -g /` for disk.
       - Returns null for unsupported OS
    6. Static `ParseInventoryOutput(string? json)`: deserializes JSON into a raw probe result, maps to HostInventory. Returns null on failure. Handles partial results (missing agents = empty list, missing disk = null).
    7. Static `ExtractVersion(string? rawOutput, string pattern)`: applies regex pattern to extract version string. Returns null if no match.
    8. Static `ResolveCapabilities(string? version, Dictionary<string, List<string>> capabilityMap)`: parses version range keys (e.g., "1.0.0+") and returns capabilities if detected version meets the minimum. Returns empty list if version is null or below all thresholds.
    9. Private `PollHostAsync`: SSH into host, execute composite command, parse output, build HostInventory (with capabilities resolved from agents.json), serialize to InventoryJson, save to HostEntity in DB, call `_cache.UpdateInventory(hostId, inventory)`.
    10. Load agents.json config using `IConfiguration` binding or direct file read (use IConfiguration.GetSection("Agents").Get<List<AgentConfig>>() if bound, otherwise read file from `config/agents.json` path). Define `AgentConfig` record: `(string Name, string VersionFlag, string VersionPattern, Dictionary<string, List<string>> Capabilities)`.

    IMPORTANT: Do NOT use jq in Linux/macOS probe scripts -- use printf string concatenation for JSON building to avoid remote host dependency. Follow research open question #1 recommendation.

    IMPORTANT: Register the service as singleton first, then as hosted service (per research open question #3):
    ```csharp
    builder.Services.AddSingleton<HostInventoryPollingService>();
    builder.Services.AddHostedService(sp => sp.GetRequiredService<HostInventoryPollingService>());
    ```
    This allows on-demand refresh endpoints to inject the service instance directly.
