**SessionDetail.razor (lines 174-189):** The SSE Task.Run loop exits silently when the stream ends (session completes), leaving `_session.State` as Running permanently. The diff stats panel at line 79 checks `_session.State is SessionState.Running` and shows "still running" forever. Add a final state refresh after the ReadAllAsync loop completes. Replace the try block inside Task.Run:

```csharp
try
{
    await foreach (var evt in reader.ReadAllAsync(_cts.Token))
    {
        _events.Add(evt);
        if (evt.Kind == SessionEventKind.ApprovalRequest)
            _pendingApprovals.Add(evt);
        if (evt.Kind == SessionEventKind.StateChanged)
            _session = await Api.GetSessionAsync(SessionId, _cts.Token);
        await InvokeAsync(StateHasChanged);
    }
    // Stream ended — refresh session state so UI reflects final state
    _session = await Api.GetSessionAsync(SessionId, CancellationToken.None);
    await InvokeAsync(StateHasChanged);
}
catch (OperationCanceledException) { }
```

Use `CancellationToken.None` for the final refresh since `_cts` may be cancelled during disposal, but we still want the state refresh if the stream ended naturally.

**Program.cs (line 288 — diff endpoint):** Add prefix matching fallback when exact match fails. Replace the session lookup:

```csharp
var entity = await db.Sessions.FirstOrDefaultAsync(s => s.SessionId == sessionId && s.OwnerUserId == user.UserId, ct);
if (entity is null)
{
    // Prefix match fallback for truncated CLI IDs
    entity = await db.Sessions.FirstOrDefaultAsync(
        s => s.SessionId.StartsWith(sessionId) && s.OwnerUserId == user.UserId, ct);
}
```

Also apply the same prefix matching pattern to the stop endpoint and any other session-by-ID lookups that users interact with via CLI. Search for other `s.SessionId == sessionId` patterns in Program.cs and add the same fallback where the ID comes from user input (the session stop DELETE endpoint at line 235-239 is another candidate).

**Program.cs (CLI):** No changes needed to CLI Program.cs itself — the fix is server-side prefix matching. The CLI already displays truncated IDs and passes them to API endpoints.
