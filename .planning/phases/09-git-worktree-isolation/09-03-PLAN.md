**SshBackend.cs (lines 58-62):** Relax CanHandle to accept Auto mode when placement has already routed to this SSH node. The placement engine calls CanHandle after filtering by TargetHostId, so if we get here with Auto mode, it means the user explicitly targeted an SSH host:
```csharp
public bool CanHandle(SessionRequirements requirements, NodeCapability node)
    => (requirements.ExecutionMode == ExecutionMode.Ssh ||
        (requirements.ExecutionMode == ExecutionMode.Auto && !string.IsNullOrEmpty(requirements.TargetHostId)))
       && requirements.AcceptRisk
       && string.Equals(node.Backend, Name, StringComparison.OrdinalIgnoreCase)
       && node.AllowRiskyDirectExec;
```
This allows Auto+TargetHostId to route to SSH hosts when AcceptRisk is true. The AcceptRisk requirement remains — users must still opt in to direct SSH execution.
