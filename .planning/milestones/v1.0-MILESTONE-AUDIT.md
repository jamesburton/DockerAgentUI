## Integration Notes

### Architectural Debt (Non-Blocking)

1. **INT-ADAPTER (Low):** `IAgentAdapter.StartAsync()` is never called — `SshBackend` sends `HostCommandProtocol` directly. The adapter pattern exists architecturally but is not in the execution path. Adding a new agent type would require modifying `SshBackend` or the host daemon, not just registering a new adapter.

2. **INT-CONFIG (Low):** `ConfigResolutionService`, `ConfigLoader`, `ConfigScopeMerger` are registered in DI but unused. Skill/policy config works via `ISkillRegistry`/`ISkillPolicyService` directly.

3. **INT-APPROVAL-PATH (Info):** ROADMAP spec shows `/api/sessions/{id}/approval/{approvalId}/resolve` but implementation uses `/api/approvals/{approvalId}/resolve`. Clients use the correct path — internally consistent.

### Orphaned API Routes

- **POST /api/sessions/{id}/stop** — Legacy endpoint; clients use `DELETE /api/sessions/{id}?force=` instead
- **GET /api/agents** — Agent type discovery available but not consumed by clients
