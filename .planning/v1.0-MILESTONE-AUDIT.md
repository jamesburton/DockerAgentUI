## Cross-Phase Integration Issues

### INT-01 (Critical): Session History Deserialization Mismatch

**Requirements affected:** SESS-05, CLI-01, WEB-02
**Location:** `src/AgentHub.Service/Program.cs` lines 116-140

The `/api/sessions/{id}/history` endpoint returns anonymous objects with `MetaJson` (raw JSON string) and `Kind` (string via `.ToString()`). Both clients deserialize as `SessionEvent` which expects `Meta` (Dictionary) and `Kind` (enum). The field name mismatch causes metadata to always be null.

**Fix:** Use `EntityMappers.ToDto()` instead of anonymous projection.

### INT-02 (Critical): Session History Pagination Ignored

**Requirements affected:** SESS-05
**Location:** `src/AgentHub.Service/Program.cs` line 116

Both clients send `page`/`pageSize` query parameters. The server endpoint doesn't bind them — returns all events unpaginated.

**Fix:** Add parameter binding for `page`, `pageSize`, and optional `kind` filter.

### INT-03 (Moderate): Input Endpoint No Client Callers

**Requirements affected:** AGENT-03, AGENT-04
**Location:** `src/AgentHub.Service/Program.cs` line 148

`POST /api/sessions/{id}/input` exists but neither CLI nor Web exposes a send-input method. The trust-tier evaluation and sanitization-triggered approval flow in `SessionCoordinator.SendInputAsync` is unreachable from any client. Server-initiated approvals (from SshBackend execution) still work.
