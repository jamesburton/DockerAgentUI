## Critical Integration Gaps

### 1. Session History Response Shape Mismatch [CRITICAL]

**Requirements affected:** SESS-05, WEB-02, CLI-01
**Location:** `src/AgentHub.Service/Program.cs` lines 116-140

The `/api/sessions/{id}/history` endpoint returns anonymous objects with `MetaJson` (raw JSON string) and `Kind` (string via `.ToString()`). Both clients (`AgentHubApiClient` and `DashboardApiClient`) deserialize as `List<SessionEvent>`, which expects `Meta` (dictionary) and `Kind` (enum). The field name mismatch causes metadata to always be null.

**Fix:** Use `EntityMappers.ToDto()` instead of anonymous projection. The mapper already exists and correctly handles the conversion.

### 2. History Pagination Ignored [CRITICAL]

**Requirements affected:** SESS-05, CLI-01, WEB-02
**Location:** `src/AgentHub.Service/Program.cs` line 116

Both clients send `page`/`pageSize`/`kind` query parameters. The server endpoint doesn't bind them — returns all events unpaginated.

**Fix:** Add parameter binding for `page`, `pageSize`, and `kind` filter.

### 3. Unused Config Resolution Pipeline [MODERATE]

**Requirements affected:** AGENT-02
**Location:** `src/AgentHub.Orchestration/Config/ConfigResolutionService.cs`

Registered in DI (Program.cs line 55) but never injected or called. No API endpoint or command exercises scoped config resolution.
